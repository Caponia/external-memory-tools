# AI Memory

本目录是 {{OWNER}} 在不同 AI 工具之间共享的长期记忆库，也是相关信息的唯一真源。

最后更新：{{DATE}}

## 读取顺序

1. `RULES.md`
2. `PROFILE.md`
3. `PROJECTS.md`
4. 与当前任务有关的 `WORKFLOWS/` 文件
5. 必要时查看 `CHANGELOG.md`

只读取完成当前任务所需的内容，不要一次加载全部资料。

## 使用原则

- 当前对话中用户的明确要求高于长期记忆。
- 新旧记录冲突时，优先采用日期较新且明确确认的记录；仍不确定时说明冲突。
- 默认只读。AI 先在聊天中提出候选更新，未经明确授权不写文件。
- 用户明确授权后，只在 `INBOX/` 新建独立候选文件。
- 候选内容经用户再次确认后，才能合并到主文件并更新 `CHANGELOG.md`。
- 项目细节优先从对应工作区读取，临时过程不全部写入长期记忆。
- 平台自带记忆或 `Exports/` 快照不得覆盖本目录中较新的确认记录。

## 敏感信息

不要保存或复述密码、Token、API Key、Cookie、SSH 私钥、密钥路径、`.env`、`.dev.vars`、银行卡、证件号、联系方式、完整住址、账号标识或其他不必要的敏感信息。

可以记录“凭据保存在密码管理器中”或“凭据不进入共享记忆”，但不记录具体值和具体位置。

## 更新流程

1. AI 在聊天中提出候选更新。
2. 用户明确授权后，在 `INBOX/` 新建带日期、来源和主题的候选文件。
3. 用户确认候选内容。
4. 经再次明确批准后，更新主文件和 `CHANGELOG.md`。
5. 重新读取更新结果并完成敏感信息检查。
