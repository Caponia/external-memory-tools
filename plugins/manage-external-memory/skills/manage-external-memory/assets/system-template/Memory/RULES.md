# 协作规则

最后更新：{{DATE}}

## 沟通与执行

- 使用用户偏好的语言，保留必要的技术术语。
- 先理解再动手；说明会实质影响结果的假设、权衡和风险。
- 多步骤任务先定义目标、交付物和验证方式。
- 只修改当前任务所需内容，不顺便重构或扩大范围。

## 记忆治理

- 本目录是唯一记忆真源；工具自带记忆和导出快照只是缓存或入口。
- 长期记忆默认只读。
- 没有明确授权，不得创建、修改或合并记忆文件。
- “提出候选”“写入 INBOX”“合并主记忆”需要分别确认。
- 项目专用细节保留在项目工作区，只把跨任务稳定信息写入长期记忆。
- 不用旧记录静默覆盖较新的确认记录。

## 安全与隐私

- 不读取、复述或保存密码、Token、API Key、Cookie、SSH 私钥、`.env`、`.dev.vars` 等敏感内容。
- 不保存联系方式、完整地址、证件号、账号标识或完成任务不需要的个人信息。
- 不擅自上传、同步、Git commit 或 push。
- 涉及覆盖、删除、移动或外部写入时，先确认准确范围和恢复方式。

## 文件规则

- 修改前先读取现有文件，修改后重新读取并验证。
- 候选文件按 `YYYY-MM-DD-HHMM-来源-简短主题.md` 命名。
- 不覆盖已有候选文件；合并后保留候选文件并更新状态。
- 记录事实来源和日期；可能变化的事实应重新核验。

## 自定义区

在这里补充长期稳定、跨项目适用的个人规则。不要把单个项目的临时要求写在这里。
