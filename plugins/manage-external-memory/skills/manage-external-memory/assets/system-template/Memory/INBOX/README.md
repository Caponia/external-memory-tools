# 待确认记忆更新

## 规则

- 默认不写入。AI 先在聊天中提出候选更新，只有用户明确授权后才创建文件。
- 每个来源、每次更新使用独立文件，避免多个工具同时修改同一个文件。
- 文件名格式：`YYYY-MM-DD-HHMM-来源-简短主题.md`。
- 候选内容经用户再次确认后，才能合并到主文件。
- 合并后保留候选文件并标记状态，不直接删除。

## 模板

```markdown
# 候选记忆更新

日期：YYYY-MM-DD HH:MM
来源：工具或用户名称
状态：待确认

## 建议更新

- 内容

## 建议落点

- `PROFILE.md` / `PROJECTS.md` / `RULES.md` / `WORKFLOWS/...`

## 依据

- 对话或文件来源

## 敏感信息检查

- 未包含凭据、密钥、联系方式、完整地址或不必要的账号信息。
```
