# 管理外部记忆

把本地 Markdown 目录作为唯一记忆真源，让不同 AI 工具只通过薄入口读取它。默认只读；把“提出更新”“写入候选区”“合并主记忆”视为三个独立动作。

## 第一优先级：新手教程

当用户输入“新手教程”“小白教程”“怎么用”“使用说明”，或表达第一次使用、完全不会用时：

1. 完整读取 `references/beginner-guide.md`。
2. 直接在聊天中输出完整教程，让用户不打开本地文件也能学会。不要只给摘要、目录、文件路径或“请阅读说明书”。
3. 保留教程中的复制即用提示词、目录解释、更新流程、安全红线、常见问题和一页速查；可以根据已知操作系统替换示例路径，但不能省略关键部分。
4. 使用短句和生活化比喻；首次出现的英文或技术词立即解释。
5. 本轮只教学，不创建、不修改、不扫描任何记忆文件，也不要求用户先提供路径。
6. 输出教程后停止其他工作，让用户自行选择教程末尾的下一步。

## 选择操作

- 新手教程：优先执行“第一优先级：新手教程”，不要进入其他操作。
- 新建记忆库：执行“初始化新库”。
- 解释或盘点结构：读取 `references/schema.md`，再检查用户指定的目录；只报告完成当前问题所需的信息。
- 使用已有记忆：执行“读取已有记忆”。
- 更新记忆：执行“候选更新与合并”。
- 分享或迁移：执行“分享与迁移”。
- 检查完整性：运行 `scripts/validate_memory.py`。

## 初始化新库

1. 确认目标根目录、显示名称和需要的工具入口。支持的入口为 `codex`、`claude`、`workbuddy`、`hermes`；未指定时只创建 `codex`。
2. 读取 `references/schema.md`，向用户说明将创建的核心文件和默认只读规则。
3. 确认目标目录尚不存在。不要向现有目录合并，不要覆盖文件；需要迁移时改走“分享与迁移”。
4. 先预演：

   ```bash
   python scripts/init_memory.py --root <目标根目录> --owner <显示名称> --adapters codex --dry-run
   ```

5. 用户已经明确要求创建时，执行同一命令并去掉 `--dry-run`。仅咨询、评估或设计请求不得产生文件。
6. 运行验证：

   ```bash
   python scripts/validate_memory.py --root <目标根目录>
   ```

7. 报告实际创建路径、入口文件、验证结果和未自动完成的事项。不要声称某个 AI 已自动加载入口，除非已经在对应工具中验证。

## 读取已有记忆

1. 让用户指定记忆库路径，或从当前工作区的入口文件解析路径；不要在磁盘上广泛搜索私人资料。
2. 规范化路径：如果 `<路径>/Memory/README.md` 存在，把 `<路径>/Memory` 作为记忆目录；如果 `<路径>/README.md` 和 `<路径>/RULES.md` 同时存在，直接把 `<路径>` 作为记忆目录；两种情况都不满足时停止并报告缺失项。
3. 先完整读取记忆目录中的 `README.md`，再按其中顺序读取 `RULES.md`、`PROFILE.md`、`PROJECTS.md` 和当前任务需要的 `WORKFLOWS/` 文件。
4. 只读取当前问题所需的部分。不要读取、复述或保存密码、Token、API Key、Cookie、SSH 私钥、`.env`、`.dev.vars`、证件号、完整住址、联系方式或其他不必要的账号信息。
5. 当前对话中用户的明确要求优先；记录冲突时优先使用日期更晚且明确确认的记录。无法判断时说明冲突，不自行合并。
6. 未在当前回合验证的旧记录要标注为记忆来源；可能变化的事实应重新核验。

## 候选更新与合并

1. 先在聊天中给出候选内容、建议落点、依据和敏感信息检查；此时不写文件。
2. 只有用户明确授权“写入 INBOX”或同等清晰指令后，才在 `Memory/INBOX/` 新建独立候选文件。不要覆盖已有候选文件。
3. 使用 `YYYY-MM-DD-HHMM-来源-简短主题.md` 命名，并遵循 `Memory/INBOX/README.md` 中的模板。
4. 写入候选区不等于批准主记忆变更。只有用户再次明确批准合并后，才修改对应主文件和 `Memory/CHANGELOG.md`。
5. 合并后保留候选文件，并把状态改为“已合并”或记录未采纳原因。不要静默删除历史。
6. 每次写入后重新读取目标文件，核对路径、内容和敏感信息。

## 分享与迁移

1. 先区分结构与内容：分享模板、规则和脚本，不分享个人画像、真实项目、候选记录、派生快照或私有工作流。
2. 保持一份内容真源；不同 AI 工具只保留薄入口，不复制完整主记忆。
3. 迁移现有库时，先生成全新的空库，再逐项比较和人工确认。不要用初始化脚本指向现有目录。
4. 对不能直接读取本地文件的工具，只生成经过脱敏的派生快照，并明确标注它不是真源。
5. 不自动配置云同步、远程仓库或外部服务；这些动作需要用户单独选择并授权。

## 不可破坏的规则

- 默认只读；没有明确授权就不写入长期记忆。
- 不直接修改主记忆来代替候选流程。
- 不覆盖、删除或重建现有记忆库。
- 不把凭据或敏感个人信息写入记忆、日志、示例或测试数据。
- 不擅自 Git commit、push、上传或同步。
- 不把平台自带记忆当成能覆盖本地真源的第二主库。

## 资源

- `scripts/init_memory.py`：从模板原子化创建一个新库；拒绝已存在的目标路径。
- `scripts/validate_memory.py`：检查必需文件、未替换占位符和敏感文件名；不会扫描凭据内容。
- `references/beginner-guide.md`：用户输入“新手教程”时必须在聊天中完整展示的小白说明。
- `references/schema.md`：目录契约、文件职责、更新状态机和工具入口说明。
- `assets/system-template/`：工具无关的空白记忆库模板。
- `assets/adapters/`：Codex、Claude、WorkBuddy 和 Hermes 的薄入口模板。
